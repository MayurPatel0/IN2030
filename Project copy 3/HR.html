Headline Review

Completing checkpoints

Structure

+

The Heading Structure can be checked with Web Developer (Information > View Document Outline) or Headings Map.

Inappropriate use of heading semantics on individual headings can be checked with the Paul J Adams - Heading Bookmarklet.

Keyboard

+

Use a keyboard to navigate through the whole page. Make sure there are no keyboard traps and focus is logical. As you tab through the page, visually check that the focus indicator is displayed throughout.

Use the focus order bookmarklet to expose the elements in the focus order.

To track focus through the page, you can use Accessibility Insights.

This has an ad-hoc for testing the tab stops. You can also use the force focus bookmarklet or Focus indicator Chrome extension when it is not clear what you are focusing on.

Skip Links

+

Skip links require manually checking. Select the browser address bar and tab into the page, before you tab onto the navigation, there should be a skip link. It should become visible when you tab onto it.

When you activate it, it should prompt focus to jump over navigation. This can be tested by activating the link and then tabbing again. The keyboard focus should have jumped over the navigation.

If there are skip links that do not seem to become visible on focus, you can open the Dev Tools and use Ctrl + F to search for “Skip Links”. This can unearth inoperable skip links.
